# 如何使用 PowerShell 查看您的电脑上次重新启动的时间

> 原文：<https://lifehacker.com/use-event-viewer-to-tell-when-your-pc-last-rebooted-5459919>

没有什么比回到您的 Windows PC 时发现所有内容都已刷新更好的了。你没有打开的应用程序。你的网页浏览器标签不见了，你的浏览器本身也不见了。你在写的那份文件？希望你自动保存了，因为看起来你的系统因为某种原因重置了。

虽然我不一定能告诉你 Windows 重新启动的原因，但是很容易看到*何时*发生了重置，这可能会帮助你根据你自己所做的事情查明问题。(或者，最起码，你会很清楚是定期的 Windows 更新，还是随机的午夜崩溃。)

首先，启动 PowerShell 并键入以下命令:

`Get-EventLog -LogName System |? {$_.EventID -in (6005,6006,6008,6009,1074,1076)} | ft TimeGenerated,EventId,Message -AutoSize -wrap`

(如果那样不行，因为 Kinja 对人物比较挑剔，干脆从 [这个 Quora 答案](https://serverfault.com/a/1002420) 复制粘贴过来。)

不需要太多的调查就能知道你的系统何时关闭和启动。在上面的示例中，您将看到 EventID 6006 和 1074 在两天前(3 月 22 日)大约相同的时间发生，大约在下午 4:00。当时我关闭了我的计算机(因此事件日志服务停止，EventID 1074 发出明显的“关闭消息”)。我在那天的 5:20 打开了我的计算机，这是 Windows 在启动时报告其产品名称(EventID 6009)和事件日志服务启动(EventID 6005)的时间。

当您盯着日志时，您还可以寻找“计划外的”或“意外的”关机，这可能会为您提供另一个线索，即您的系统崩溃了，停电了，或者有其他一些莫名其妙的关闭原因。

你也可以尝试直接在 Windows 10 的事件查看器中搜索这些事件。只需打开 Windows 日志文件夹，点击系统，然后开始滚动(或过滤)上述事件 id。当你这样做的时候，把 EventID“41”放入其中，这将让你知道你的系统是否在没有干净地关闭的情况下重新启动了——这是一个指示，它可能崩溃了，遭遇了断电，或者只是有一个关闭问题。
