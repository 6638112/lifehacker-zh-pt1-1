# 请停止使用短信接收登录代码

> 原文:[https://life hacker . com/please-stop-using-text-messaging-to-receive-log in-codes-1846495160](https://lifehacker.com/please-stop-using-text-messaging-to-receive-login-codes-1846495160)

本周，一个来自 [Vice](https://www.vice.com/en/article/y3g8wb/hacker-got-my-texts-16-dollars-sakari-netnumber) 的惊人故事揭示了攻击者窃取你的短信是多么容易。他们不需要接触你的手机；他们甚至不需要你的SIM卡。他们只需要支付一笔微不足道的费用，让一个VoIP批发商相信他们是经销商(也是小事一桩)，并签署一份表格，保证他们可以将你的号码发送的信息发送给另一个号码。

我不会深入探讨这种方法的本质，这种方法可以用来将你的短信从你的手机中路由出去，但事实是它曾经是(现在也是？)这么容易做到，*和*以至于你收不到一个批准查询，甚至收不到一个通知，这是不和谐的。

虽然我确信许多这种企业级短信服务正在加强其安全性，但攻击者只需找到一个不与号码的实际所有者验证这种变化的服务，就可以永别了，收到短信。这包括当你在一个未知设备上登录一个账户时，用来验证你是不是你自己的验证码。

我们以前说过，我们将继续说下去，直到所有的网站和服务最终听取:仅仅使用一条短信或两步认证来保护一个人的帐户免受未经授权的访问是不够安全的。只要有可能，您应该使用专用的双因素身份验证应用程序，该应用程序*要求*物理访问您的硬件(通常是您的手机)来完成帐户的登录过程。短信并不像你想象的那么安全。虽然你自己可能永远不会成为短信劫持的受害者，但本周的新闻显示这并非不可能。

不太可能有人得到你的实际智能手机，找到一种方法绕过你现有的安全机制(触摸或面部识别)来解锁它，通过你在特定的2FA应用程序上设置的任何二级安全措施(如PIN码)，然后*然后*用它来侵入你的账户。到那时，他们可能要么放弃，要么你已经重置了你的2FA，并为你的关键账户设置在一个新的设备上，使旧的代码完全无效。

你不必注册一个监控工具，在你的电话号码被发送到其他地方时提醒你。(充分披露:前述媒体撰稿人是一家此类公司 的 [首席信息官，好的)。然而，你可能只是想这样做，因为有很多服务*仍然*使用文本信息，而且仅仅是文本信息，来给你发送登录代码。](https://okeymonitor.com/)

如果您的医疗服务提供商、游戏网站或其他网站不允许您使用双因素认证，而只允许您使用两步*认证，您将无能为力。选择一个强有力的，独特的密码，用一个伟大的密码管理应用程序锁定它，并期待最好的结果。此外，不要对你的安全问题使用明显的答案； [那些也应该是“密码](https://lifehacker.com/use-your-password-manager-for-security-answers-too-1829498257) ”，你应该像追踪其他密码一样追踪它们。*

最后，不要*而不是*使用两步认证，如果你只有这一步认证的话。虽然它不是100%安全的，但启用它并迫使某人跳过额外的关卡来侵入您的帐户要好得多。如果你能在组合中加入一点额外的安全性，不要仅仅依赖你的登录+密码组合。

还有更极端的方法，比如使用一个与你实际电话号码完全不相关的专用登录码。(想到了谷歌语音；你可以让它只给你发邮件，你可以用双重认证锁定你的谷歌账户。)虽然这可能无法阻止某个人随意劫持那个号码，但至少可以帮助你免受有针对性的攻击。嗯，*更安全。*安全不好玩吗？
