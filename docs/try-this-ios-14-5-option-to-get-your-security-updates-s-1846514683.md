# 尝试此iOS 14.5选项以更快获得安全更新

> 原文:[https://life hacker . com/try-this-IOs-14-5-option-to-get-your-security-updates-s-1846514683](https://lifehacker.com/try-this-ios-14-5-option-to-get-your-security-updates-s-1846514683)

从我们目前的四个开发者测试版来看，iOS 14.5应该就在眼前了。在这一点上，我不会嫉妒任何想在他们的主要手机上尝试更新的操作系统的人(只要你选择更稳定的 [公测版](https://beta.apple.com/sp/betaprogram/) )。)但是,对iOS 14.5有一个小小的改变，以唤起那些在测试版上的 *或*那些急切等待新版iOS的，其中是新的，略有修改的软件更新体验。

****大多数人应该让这些功能保持原样，因为让拥有一台自动更新最新主要功能更新和更频繁的安全更新的设备是件好事。然而，苹果的新处理方式似乎暗示，它甚至可能开始为无法获得最新主要iOS版本的设备提供安全更新——[纯粹是猜测](https://9to5mac.com/2021/03/15/ios-security-fixes-could-soon-be-delivered-separately-from-other-updates-beta-code-suggests/) ，但我们希望能够在安全更新准备就绪后尽快获得安全更新，而不是苹果将它们绑定到一个iOS版本，即在旧设备 ce s上可能无法访问****

****到目前为止，该功能的一个奇怪之处是，“安装安全更新”选项与“下载新更新”绑定在一起，所以你目前不能告诉你的iPhone，你*只*希望它自动下载安全更新(并且你宁愿自己管理面向功能的更新)。不过，我认为这是苹果的意图，因为该公司为你提供自动下载常规iOS更新的选项，但在你空闲时有选择地下载安全更新，这将会很奇怪。后者是比前者更优先的。****

****不过，我刚刚看到这个选项出现在我的设备上，所以苹果很有可能仍在解决这个问题。请留意此功能以及您的设备可能希望您安装的任何补充更新。如果你还没准备好跳到另一个版本的iOS，至少你可以在等待的时候堵住设备上的漏洞。****
