# 什么是 IPFS，为什么它在 Brave 的网络浏览器中很重要？

> 原文：<https://lifehacker.com/what-is-ipfs-and-why-does-it-matter-in-braves-web-brows-1846103479>

Brave 现在是第一个具有 [集成星际文件系统(IPFS)支持的网络浏览器。](https://brave.com/ipfs-support/) IPFS 是一种点对点超媒体协议，与我们已经使用了几十年的 HTTP(超文本传输协议)和 HTTPS(安全套接字层上的超文本传输协议)协议相比，它提供了一种不同的构建和使用互联网的方式。

对于 Brave 用户来说，这是一个显著的变化，它影响了浏览速度、连接稳定性、隐私甚至立法，但如果其他浏览器效仿 Brave，它甚至可能影响互联网的未来。在我们深入了解为什么 IPFS 支持如此重要之前，让我们解释一下它是如何工作的，以及它与 HTTP/HTTPS 的不同之处。

### 什么是 IPFS？

当您使用 HTTP 或 HTTPS 时，您的浏览器使用 URL(统一资源定位器)从中央服务器访问网站。您与服务器的物理距离会影响使用的带宽和加载页面的时间。

IPFS 通过网络分发网站数据，而不是 URL 和服务器。使用 URIs(统一资源标识符)访问数据。你可以在 Brave 的公告帖 上了解更多关于 IPFS 如何运作 [的信息，但简单来说，这类似于 BitTorrent 和区块链。每台计算机或移动设备——被称为“节点”——临时存储一个网站的数据。因此，每当你通过 IPFS 访问一个网站时，你都是在从网络上的其他邻近节点加载数据。如果用户不希望充当本地节点，他们也可以通过“公共网关”访问 IPFS 内容。](https://brave.com/ipfs-support/)

### IPFS 的利与弊

IPFS 分散托管最直接的效果是减少了加载网页的时间。由于您是从本地节点而不是远程服务器访问数据，加载时间和带宽需求可能会下降，文件传输和流速度可能会快得多。如果 IPFS 起飞，它可以减少，甚至完全消除对集中式服务器的需求。如果服务器离线，所有用户的网站将不再崩溃，因为它的数据将在整个网络中传播。

分散托管也使得自上而下的审查——比如来自政府或网络托管公司的审查——变得更加困难。不过，这是一个令人担忧的问题。在言论自由不受保护的国家，削弱独裁内容法可能会让重要信息更容易被找到。然而，这也意味着有问题的网站将更加难以关闭。我并不是说“压制”对立的政治观点，因为没有一个集中的服务器来托管有问题的网站或内容，所以像贩卖人口、虐待儿童和恐怖活动这样的真正危险将更难发现和消除。

无论你是在 Brave 的 IPFS 网络中充当本地节点，还是只是加载 IPFS 的内容，都存在隐私问题。如果你是一个节点，网络会给你一个唯一的 ID。这个 ID 是散列的，但是其他用户可以看到，并且可以用来查看其他人正在托管和访问什么。如果其他用户正在访问您当前托管的 IPFS 数据，您的设备和个人网络的资源也会被使用。

你也可以选择*不*作为一个节点，简单地通过公共网关访问 IPFS 内容，但是公共网关可以查看和记录你的 IP 地址。您可以在 Brave 的 IPFS 安全支持页面 上阅读有关这些问题如何具体影响 Brave 的更多信息，并找到减轻风险的提示— [。](https://support.brave.com/hc/en-us/articles/360051406452-How-does-IPFS-Impact-my-Privacy-)

如果 IPFS 能完全取代 HTTP/HTTPS，这些问题都需要解决。目前，Brave 使用 IPFS 和 HTTP/HTTPS 来提高速度和稳定性——尽管它只会影响配置为 IPFS 托管的内容。

### 在《勇敢》中使用 IPFS

IPFS 工作在勇敢 1.19 或更高版本。要启用它，请转到**设置>扩展**，然后在**“解析 IPFS 资源的方法”下选择您喜欢的 IPFS 方法“本地节点”**将使用您的本地节点，**【网关】**使用公共网关，**【Ask】**让您在每次访问 IPFS 内容时进行选择。**【禁用】**关闭 IPFS。

(注意:如果您使用 Windows Security 或其他程序设置了防火墙，如果您使用“本地节点”选项，可能会要求您配置 Brave 的 IPFS 节点。)

启用**“IPFS 公共网关回退”**允许您加载 IPFS 内容，即使您的本地节点不工作。然而，如前所述，这使得您的 IP 地址可跟踪。

打开**“IPFS 伴侣”**为 Brave 添加一个扩展，用于控制你的本地节点设置，并查看其连接状态。如果你不想使用 IPFS 伴侣插件，你也可以配置 Brave 的本地 IPFS 节点设置，方法是转到本页的`brave://ipfs.`，你可以打开或关闭你的本地节点，并查看当前连接了多少个对等节点。

连接所有设备后，您就可以开始访问 IPFS 内容了。这样做需要一个 URI。在更多的用户、网站和浏览器采用 IPFS 支持之前，这些都很难实现，但是 Brave 有一个例子，URI 为文森特·梵高加载了一个维基页面

只需复制并粘贴到 Brave 的导航栏来加载页面。请注意，由于 IPFS 的隐私权衡，IPFS 在使用 Brave 的内置 Tor 隐私浏览时被禁用。
