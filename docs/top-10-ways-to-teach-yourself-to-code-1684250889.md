# 14种自学编程的好方法

> 原文:[https://life hacker . com/top-10-ways-to-teach-yourself-to-code-1684250889](https://lifehacker.com/top-10-ways-to-teach-yourself-to-code-1684250889)

如果你想学习如何编码，向这个巨大的宇宙迈出第一步可能看起来是一项艰巨的任务。这里有一个大秘密:有大量免费(且便宜)的资源可以用来给自己提供所需的所有帮助，自学新技术，并使这个学习过程变得有趣和令人兴奋——至少像编码一样令人兴奋。

无论您是一个希望进入科技行业的成年人，一个希望学习最新语言的学生，还是一个只想了解软件和服务如何工作的业余爱好者，您需要的只是一台计算机和互联网接入来开始您的编程之旅。但是在你飞跃进入*矩阵*之前，这里有我们最好的建议和资源，让你有个好的开始。

花几分钟(或一天)想想你想学习一门编程语言的原因——真正的原因。对自己诚实。你是否在努力学习获得晋升的最低要求？你想在职业上有大的改变吗？你想创造下一个伟大的应用吗？通过对你的各种智能设备 [进行编程来做一些很棒的事情](https://www.youtube.com/watch?v=ebJFIKMN_Eo) ，让你的室友兴奋不已？

您的答案可以帮助确定您应该掌握哪种编程语言，以及您的目标可能需要什么样的承诺(时间和金钱)。例如，如果你的梦想是创造下一个伟大的操作系统或PhotoShop的杀手级替代品，你将受益于教你C++的正式计算机科学教育，以及更复杂的主题，如数据结构、算法和内存分配。

另一方面，如果你是一名职业生涯中期的专业人士，希望过渡到技术行业，那么短期的编码训练营可能比举债攻读第二学位更有意义。如果你想做的只是建立网站或把你的Raspberry Pi推向极限，互动教程和免费在线课程的结合可能足以让你去做。

一旦你明白了你为什么要编码，你就可以更容易地确定你应该学习哪种编程语言。虽然没有单一的“最佳”编程语言可以学习，但有些语言比其他语言更易于用户使用。HTML和CSS被认为是进入编码世界最容易的入口点，但是它们只对开发基本的网站有用

对于更多涉及支付系统或数据库的交互式网站，你需要知道Javascript、PHP、SQL和Python来让所有的组件唱歌。移动应用程序制造商使用Swift或C制作iOS应用程序，使用Java或Kotlin制作Android应用程序。视频游戏开发者将转向Unity，甚至C#，来实现他们的游戏创意。

仍然不能决定你的入门编程语言？大多数专业人士建议学习Python、C#或JavaScript，因为它们提供了最广泛的实用性和职业灵活性。为了帮助你下定决心，看看这张伟大的信息图，它比较了几种流行的编程语言。

最重要的是，开始学习一些东西。一旦你熟悉了一种编程语言，你就能更快地学会下一种——就像学习一种新的乐器或外语一样。

如果你想更好地控制你的学习进度(或者不想独自学习)，在线编码课程可能比面对面的编码训练营更好。然而，有许多不同的在线课程教授相同的编程语言，很难找出哪一个真正值得你花费时间和金钱。

如果你不介意独自练习，可以看看Odin项目，这是一个免费的选项，汇集了一些最好的开源内容，用于将编码新手转变为程序员。

想要更多的一对一辅导和职业准备，而不做全职训练营？尝试报名参加 [Udacity](https://www.udacity.com/) 或 [Treehouse](https://join.teamtreehouse.com/techdegree/) 的订阅模式认证课程，在那里你有机会向导师寻求帮助，而不是独自忍受你的编码错误或问题。你不可能比Udemy的19美元的编码课程做得更好了(甚至更少，如果有销售的话)——即使是全职的训练营和计算机科学学生也报名参加这些课程来补充他们的学习。

无论你完成了多少课程，许多初学者仍然很难应用他们相对基础的知识。为此，许多人推荐来自 [Watch和](https://watchandcode.com/)Code的免费实用JavaScript课程，该课程围绕一个你不断迭代的项目。

不要过度专注于学习一门特定的编程语言，你也可以学习用计算机能够理解的方式解决问题。换句话说，提高你在模式识别、算法和抽象概念方面的技能。还有隐语，比如循环，在你使用的任何语言中都会出现。你对这些原则理解得越好，学习下一门语言就越容易(从而设计出更好的产品或项目)。

由于有了互联网，你不必参加四年制的大学课程来学习计算机科学的基础知识；许多大学水平的课程在网上免费提供(或象征性收费)。

哈佛的CS50x计算机科学导论怎么推荐都不为过。你可以通过 [edX](https://courses.edx.org/courses/course-v1:HarvardX+CS50+X/course/) 免费旁听课程，完成所有作业获得证书，也可以按照自己的进度进行，观看 [Youtube](https://www.youtube.com/channel/UCcabW7890RKJzL968QWEykA?sub_confirmation=1) 上发布的所有讲座。你可以用这十五门在线课程(其中许多也列在我们的 [Lifehacker U](https://lifehacker.com/tag/lifehacker-u) 系列中)来构建自己的DIY大学水平计算机科学程序。

学习编码的最好方法可能是让你近距离接触一些枯树——一本你可以从头到尾阅读的真正的书。在一个完美的世界里，这将比在网站上从一个主题跳到另一个主题给你一个更全面的编码介绍。

你或许可以在GitHub上找到任何你想学习的语言或框架的电子书，但许多web开发人员对乔恩·达克特(Jon Duckett)丰富多彩的系列 深信不疑，从前端到后端的web开发都是如此。

作为一名自学成才的程序员，你也可以从提供更广泛职业建议的书籍中受益。你会改善你的思维和习惯，这些特质可以应用到你决定学习的任何语言中。史蒂夫·麦康奈尔的900页 [代码全集:软件构造实用手册(第二版)](https://www.amazon.com/Code-Complete-Practical-Handbook-Construction/dp/0735619670/ref=cm_cr_arp_d_product_top?asc_campaign=InlineText&asc_refurl=https://lifehacker.com/top-10-ways-to-teach-yourself-to-code-1684250889&asc_source=&ie=UTF8&tag=kinjalifehackerlink-20) 被认为是最佳实践的权威指南，从项目设计到调试代码，一切都有数据支持的建议。

这个庞然大物的必然结果是“鲍勃大叔”罗伯特·马丁的《干净的代码:敏捷软件工艺手册》 。Martin认为应该高效地编写代码，以便于测试和伸缩，这是许多现代开发团队的必读之作。

因为编程实际上是解决问题，所以你会欣赏Jon Bentley在 [【编程珍珠(第二版)](https://www.amazon.com/Programming-Pearls-2nd-Jon-Bentley/dp/0201657880/ref=cm_cr_arp_d_product_top?asc_campaign=InlineText&asc_refurl=https://lifehacker.com/top-10-ways-to-teach-yourself-to-code-1684250889&asc_source=&ie=UTF8;&pldnSite=1&sa-no-redirect=1&tag=kinjalifehackerlink-20) 中分享的思考过程和“信封背面”的解决方案。每章都以一些经常在技术性工作面试中使用的挑战结尾，所以让这本书成为你工作准备的一部分。

让我们面对现实吧:阅读屏幕上或书中的代码行可能看起来像胡言乱语。当你看到一个实际的例子时，这个抽象的概念突然变得非常有意义。这就是你可以在网上找到的交互式编码教程的美妙之处。

例如， [FreeCodeCamp](https://www.freecodecamp.org/) 不仅在你的网络浏览器中将编码概念分解成小块，它还将每个概念与一个相关的练习配对，你必须在进入下一个主题之前解决这个练习。这样，你就不能直接跳到有趣的事情上；这个网站真的迫使你在学习的时候调试你的代码。

编写游戏代码也是一种尝试编程的有趣方式。如果你有一个小时可以打发，可以试试简单的 [小时代码](https://code.org/learn) 游戏。此外，许多学校已经使用 [《我的世界》:教育版](https://education.minecraft.net/get-started/).) 来教孩子们用代码块甚至JavaScript编程的基础知识。(如果你有Office 365教育帐户，可以免费下载这个版本的《我的世界》。)

想要一个更复杂的游戏开发教程放在你的简历上吗？如果你不想整天打砖块，那么CodinGame 可能更适合你的年龄。

谁说成年人不能从让孩子迷上编码的STEAM/STEM玩具和视频游戏中学习呢？毕竟，这些产品旨在教授编码逻辑和语法，而不会让容易分心的孩子感到无聊，因此即使是成年程序员也可能会觉得它们有趣且有教育意义。

如果你想更好地理解硬件和软件之间的关系，那么你会喜欢 [派珀计算机套件2](https://www.playpiper.com/products) ，它让你建立一个树莓Pi驱动的计算机。使用这款DIY笔记本电脑，您可以通过其定制的《我的世界》故事模式挑战学习编码，使用拖放块状语言学习物理计算，或者只是从预安装的课程中学习一些基本的Python。

虽然你可能无法从*星球大战*中创造出自己的BB-8机器人，但你可以学习用 [Sphero的螺栓](https://www.sphero.com/sphero-bolt) 编写一个。使用配套的应用程序，你可以使用名为Scratch的拖放编码块来控制你的droid球，或者升级到JavaScript来编程更高级的动作。

你有类似亚马逊Echo的智能家居设备吗？您可以通过创建定制的迷你程序来测试您的编码技能，以便从您的设备的数字助理中获得更多功能。亚马逊的Alexa可能已经知道许多基本的语音命令“技能”，比如阅读最新的新闻标题，但你可以通过在Node.js、Java、Python、C#或Go中编码来教她更复杂的任务 。(或者，如果你想从更简单的东西开始，试试更简单的Alexa [技能蓝图](https://blueprints.amazon.com/?asc_campaign=InlineText&asc_refurl=https://lifehacker.com/top-10-ways-to-teach-yourself-to-code-1684250889&asc_source=&tag=kinjalifehackerlink-20) 网站。)

你对编码的兴趣可能更有限——例如，你可能只想学习足够多的Python来让你的Raspberry Pi做很酷的项目。如果是这样的话，你可以通过查看其他人在网站 上分享的许多项目 [来采取DIY的学习方法。](https://projects.raspberrypi.org/en/projects)

通过重新创建现有的项目，你将了解更多关于你的 [高度可定制的](https://lifehacker.com/the-raspberry-pi-4s-most-interesting-quirks-1835871780) 设备的内部工作原理。你甚至可能会发现自己受到启发，去创造新的方法来使用你的迷你电脑，并进一步钻研代码世界。(你甚至可以通过Coursera报读UC Irvine的 [树莓Pi平台和树莓Pi平台的Python编程](https://www.coursera.org/learn/raspberry-pi-platform) 课程，如果你想要一点入门帮助的话。)

有时候，看着别人告诉你怎么做更容易。如今，你可以找到各种各样的教育视频，几乎涉及任何与编码相关的主题:前谷歌员工分享编码采访的解决方案，YouTubers直播他们的编码马拉松，甚至编程老手向你展示如何用任何语言解决特定错误。

不管你完成了多少证书和编码研讨会，或者你学习了多少编程语言，你的编码技能的证明将会在你的编程项目中。虽然你的个人项目不必像创建下一个谷歌地图那样雄心勃勃，但它应该是你想要全天候工作以不断改进和扩大其范围的事情。

从小处着手，但要从大处着眼。您的项目应该包括您目前工具箱中的技能，但是您还应该有一个将愿景变为现实所需的未来功能和技能的计划。如果你没有在一些项目中应用你的编码技能，你甚至不会被考虑成为专业的编码人员——所以开始计划和练习吧。(我们的建议？使用你的编码印章为游戏之夜 建造 [一个惊人的设置。)](https://youtu.be/NvcG9Tt3WxM?t=80)

正如我们的一位编程教授所说的那样，编码就是细节，这就是为什么你需要“庆祝小胜利”。让每个元素独立工作需要实践，还需要不断的测试来确保每一行代码都能与其他代码一起工作，不会出错。如果你没有做好看似很小的事情，比如关闭一个HTML标签，你就只能调试一个简单的语法错误，而不能写出更加令人印象深刻和复杂的代码。

你需要对自己有耐心；不要指望仅仅经过几个月的学习或几个训练营就能写出下一个堡垒之夜。对于一个初学者来说，在网站上编写一个没有错误的婚礼RSVP表单，或者制作一个简单而正确的数字预测游戏，已经是一个很大的成就了。如果你遇到困难，不要放弃。你绝对应该在任何你能找到的地方寻求帮助，但也要知道什么时候该休息一下以避免精疲力尽。然后努力，再努力。

这是我们最好的编码建议:如果你不知道为什么你的代码坏了，你可以在网上寻找解决方案。毕竟，你可能不是第一个犯错误的人，互联网上肯定有人已经找到了解决你问题的方法。只要将你的错误信息“复制并粘贴”到谷歌(或你首选的搜索引擎)中，在整个短语周围加上一对引号，这样你就不只是在搜索关键词，然后点击“回车”希望这个小技巧能引导你找到正确答案。

如果你仍然有问题，你可以在开发者论坛上发表你的问题，比如 [栈溢出](https://stackoverflow.com/) ，Reddit的 [编程子编辑](https://www.reddit.com/r/programming) ，或者 [GitHub](https://github.community/) 。给朋友或网络陌生人打电话不会有坏处。

当您对他人的代码进行逆向工程，测试每一行代码以了解其工作原理时，您会对全局有更好的理解。多亏了大量的开源代码，你可以通过检查别人的(完美的)作品来了解任何东西。只要记住，如果灵感来了，你改进了你正在摆弄的程序的一部分，就把你的代码分享给社区。

你也许可以通过GitHub找到一些有用的在线导师，或者在当地的编码聚会或黑客马拉松上遇到其他资深开发人员。一旦你获得了一些编程经验，你也许能够回答其他人的问题，或者甚至把你学到的东西教给新手——这是检验你是否真正了解自己的一个很好的测试。

编码训练营可能会有争议:他们会给你一个快速介绍和大量技能的体验，但他们可能不是你获得一份全新谷歌工作的黄金门票。

如果你想成为一名专业的全职开发人员，一个密集的面对面的编码训练营可能会帮助你，特别是如果你在一个有真人激励你的结构化环境中学习得最好。然而，这些类型的训练营往往是最昂贵和最耗时的:光是学费，加上毕业前几个月的生活费，你就要花15000美元以上。

幸运的是，像 [熨斗学校](https://flatironschool.com/) 和 [格蕾丝·赫柏项目](https://www.gracehopper.com/) 这样的机构提供奖学金或延期学费，使这些训练营对少数族裔和女性科技人员等代表性不足的群体来说更实惠。查看 [课程报告](https://www.coursereport.com/) 了解许多在线和现场训练营的全面分类——包括来自校友的详细评论，这可以帮助你决定训练营是否会帮助你实现编程梦想。
